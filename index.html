Getty Center

Clarence Wang

2019-07-30

​ 来美国正好一个月了，之前总想写点这边的生活，不过总是因为各种原因鸽了。

​ 今天学校组织我们去Getty Center，学校东北边的一个私人博物馆。早上9点钟从学校坐shuttle出发，差不多一个小时到达目的地。首先自然是进门安检，不过美帝的安检连个金属探测器都没就不说了。由于Center实际上是在山上，所以进门后还需要搭乘Tram(一种轨道电车，无人驾驶的)。在Tram上可以看到公路上来来往往的车辆还有对面建造在山上的住房。这种景观在国内基本上很难见到。

​ 乘坐tram很快就能到达展厅。展厅分为四个部分，东西南北Pavilion，大部分都是中世纪的绘画，还有一些雕塑，基本上都和宗教有关，来自欧洲各个国家。由于对这些知识实在是缺乏了解，我只能走马观花看个表面。令我印象深刻的只有几幅比较逼真的风景画。实际上，Getty Center还免费提供了自助讲解的仪器(iphone4s改造的)，很多展品都有一个编号，手机输入编号就能听到相关的介绍。不过由于听力水平有限，只能听懂一小部分。

​ 印象最深的还是Garden吧，可以说是Getty家族的后花园了。这里能够看到洛杉矶的全景，不过今天似乎有雾霾。在池子前留下了今天唯一一张游客照。

​ 似乎可写的东西并不是很多。主要还是自己艺术知识太过于匮乏，难以把握好这个欣赏艺术品的好机会。

写于19年G4之后

Clarence Wang

2019-06-08

​ 105-92，金州勇士队输掉了总决赛的第二个主场，也可能是甲骨文球馆的最后一场比赛。勇士队已经大比分3-1落后，在之前的NBA总决赛历史中，33次3-1落后的球队只有1支最后完成了逆转。勇士的翻盘只剩下了理论上的可能性。五连总决赛，四年三冠的勇士王朝在或许真的已经走到了尽头。

​ 今天的失利的确让作为勇士球迷的我十分沮丧。但也的确心服口服，面对多点开花，冷静地可怕的猛龙，勇士能够拿下比赛的几率实在是太低。然而，这支球队值得我们的尊敬。汤神和鲁尼带伤复出，仍拿出了不错的表现。库里上一场拼尽全力拿下季后赛职业生涯新高的47分，这一场虽然疲倦不堪，但仍然坚持作战。很多人会将失败归咎于部分球员的表现。的确，总会有人表现不如预期。你可以说麦金尼基本功不好，罚球都丢，但他态度足够积极，拼抢足够勇敢。作为拿着球队底薪的人，你无法苛求太多。你可以怪罪一哥李文发挥不好，但是作为老将，他们已经付出了所有。

​ 有人说，这次总决赛是库里证明自己，拿下fmvp最好的机会。现在看来，他可能失去了这次机会吧。作为球迷，自然少不了遗憾。我是多么希望今年他能够拿下那该死的fmvp奖杯，让那些黑他的人闭嘴。现在看来，虽然希望落空，但他的表现已经足够出色。在杜兰特受伤后，手指脱臼的情况下，淘汰了火箭和开拓者，带领残阵和猛龙拼搏，他真的做到了能做的一切。作为身体素质一般的球员，在关键比赛、关键时刻总会遭遇疯狂的防守，他已经表现的比绝大多数球员都好。相信正常的球迷都能够看到这些，至于那些黑子，无论如何，他们总是能够找到理由去抹黑别人。

​ 可以说，库里是我看NBA的原因。在赛场上，他奋力拼搏，带领球队前行；赛场下，他是一个好丈夫、好父亲，很少传出其他人那样的绯闻。他活出了很多人想要的人生。成名路上并不是一帆风顺，他的身材并不突出，因为“只会投三分”而一直被低估，尽管在15年拿到了总冠军之后。我也会关注体育论坛上的某些帖子，帖子里总是爱给球员排名。开始时，我也十分在意人们对他的评价，慢慢的我也释然了。他永远只会有球场上的表现回应别人的质疑。的确，人活在世上，应当为自己而活，别人的看法也只是别人的看法。库里是一个独特的球员，他为了团队牺牲了很多个人的利益，我想这也是很多人喜爱他的原因。

​ 当然，竞技体育推崇成王败寇。人们总是渴望弱者逆袭，其实在我看来，缺少杜兰特，多人伤病的勇士才是弱势的一方。不过这些都不重要了。看了几年的NBA，我反而越来越感觉到媒体的可怕。很多东西都是媒体渲染而来，他们总是喜欢吸引别人的眼球从而获得更多的利益。正如赛季初考辛斯加盟勇士，几乎所有的媒体都极力报道，鼓吹五个全明星的勇士，NBA已经大结局。但是，这所谓的五星实力究竟有多少，只有真正的勇士球迷才清楚。现在，五星勇士输了，成为人们笑柄，媒体又迎来了一系列的流量，他们的目的已经达到。NBA到底还是商业联盟，金钱至上，利益至上，场上场外总是有非体育精神的因素干扰，使其并不能做到完全的公平。经历过这一切，我感觉自己对这些各种各样的讨论已经不那么在意了，我们无法控制别人的想法。我只想从这些脱离开，享受篮球的魅力，体育精神的魅力。

​ 尽管勇士输了，我仍然会支持这支球队，支持库里。对于其他人来说，你们可以不喜欢这支勇士，但还是请尊重他们。

中微结课

Clarence Wang

2019-06-01

生活

​ 今天中微结课了。

​ 中微应该是这学期出席频率最高的经双课了。老师是吴星晔，清华经管本科，哥大经济学博士毕业不久。吴老师数理基础非常扎实，上课所用的模型基本都用数学语言很好的表示出来，比课本要形式化的多；另外，他也经常讲授经济学的思维方式，而不是仅仅介绍枯燥的模型。

​ 中微的主线是为什么要实行市场经济，从单个的消费者问题出发，进而引入生产和均衡的概念， 讨论了市场如何达到均衡，以及在均衡下是Paretto Efficient的。当然，这些在实际情况下都是很难实现的。后面几节课讨论了Market Power, Externalities和Asymmetric Information这几种导致市场失衡的原因。

​ 很多的经济学现象都可以用外部性来解释。比如我早饭去食堂吃了一个包子，我影响了做包子的人，卖包子的人，还导致了别人少吃了一个包子，这也是一种外部性。但是由于我为包子付了钱，外部性被内部化了。市场经济正是一种将外部性内部化的一种目前来看最佳的方式。当然，也不排除其他方式，比如共产主义社会。甚至不同国家对言论的管控力度也可以用外部性来解释。如果认为言论有正的外部性，应当减少监管；否则应该增加监管。

​ 信息不对称也能对经济造成比较严重的影响。如果卖方知道的信息比较多，会导致质量高价格高的卖房退出市场，比如二手市场的lemon和plum；如果买房信息较多，那些保留价格较低的买方将因为价格过高无法得到服务，比如保险市场。

​ 因为外部性和信息不对称，需要政府通过法律法规作出调节。

​ 中微还是非常有意思的一门课。

我的Vim配置

Clarence Wang

2018-08-15

1. 基础配置

基础配置主要在根目录中的.vimrc中设置（没有则创建），vim .vimrc打开，添加命令。

常用的配置诸如:

  • set number “设置行号
  • set tabstop=4 “设置tab

更多的详见.vimrc

2. 添加插件

我用的是Vundle对插件进行管理。在Vundle中，插件的安装十分方便。这里对于Vundle的讲解十分详细，基本照做就行。

可以添加自己喜欢的插件，YouCompleteMe是一款十分实用的代码补全插件，支持C, C++, java多种语言，用Vundle安装即可。

3. 为不同的语言设置高亮

设置完1和2之后，我的java和c++高亮还是不够完善，不能支持类名和部分变量名。Google后发现可以在.vim/syntax下添加文件进行高亮。比如若想要配置c++的高亮，可以添加cpp.vim进行个性化的设置。各种语言语法高亮可参考我的Github。

4. 一些其他的插件：

  • airline 底部状态栏
  • Nerdtree 目录树
  • fugitive 配合airline使用可显示git分支，支持gitcommit, push等命令

均可通过Vundle安装

3. Soup Servings

题意：A, B杯中都有N ml的汤，每次进行四种操作，问A比B先倒完加上同时倒完*0.5的概率是多少

解法：难点在于N很大时，直接返回1就行了，原因是A几乎总是比B倒的多。N小的时候使用记忆化搜索

4. Chalkboard XOR Game

题意：黑板上有一串数，A和B轮流擦去一个数（A先）。如果某个人操作完之后所有数的异或和0，他就输了。如果在操作前异或和已经为0，那么他赢。问A，B谁会赢。

解法：考虑开始时黑板上数异或和不为0的情况（否则A直接赢了）。当A有偶数个数时，A总是能够移动。否则另S = x1 ^ … ^ xn != 0。当A无路可走时，任意xi，S^xi = 0。那么(S ^ x1) ^ (S ^ x2) ^…^ (S ^ xn) = (S ^ S ^… ^ S) ^ (x1 ^… ^ xn) = 0 ^ S != 0。矛盾。因此只需要判断奇偶性即可。

3. Champagne Tower

题意：玻璃杯按杨辉三角排列，向杯中倒入香槟，如果满了往下面的两个杯子分别流入多余的1/2，问倒入一定量的香槟后某个特定位置杯子中有多少香槟。

解法：一开始我想的是把每一层所有的杯子倒满，之后处理剩余的部分。后来发现剩余的部分并不是像我想的那样两边是中间的一半。这题应该采用递推模拟的方法，假设所有的香槟一下倒进去。减去1，剩下的部分给下面的两个杯子即可。

4. Smallest Rotation with Highest Score

题意：一个长度为n的数组A，对于每个位置，A[i] <= i得一分，因此可以得到数组的总得分。然后对数组进行shift操作，即将最后一个元素移到第一个位置，其他元素顺次后移。再次计算总得分。如此操作，问shift一轮之后，哪个位置的总得分最高。

解法：事实上，每个位置的元素是相对独立的，每个元素总是在特定的时候进入无法得分的区域，在另个时候出去，这个只与该元素首位置有关，与其他元素没有任何关系。所以我们只需记录每个元素i进出的时间，分别为good[i]和bad[i]，然后计算出初始的score。每次shift加上good[i]-bad[i]（就是该时刻进去的元素减去出去的元素），一轮遍历找到最大值。注意0和n-1的特殊性。

LeetCode793 Preimage Size of Factorial Zeroes Function

Clarence Wang

2018-03-04

题目意思：输入K，找出阶乘中结尾刚好有K个0的自然数数目

题目链接

思路：每个自然数的结尾0数是一个单调不减的序列，二分查找K和K+1（返回最右侧）的位置再进行做差即可。

一、如何计算n!结尾后面0的个数？

用f(x)表示n!结尾后0的数目。结尾之后的0经过因式分解之后必然为2和5，n! = 2^a * 3^b * 5^c *…。则f(x) = min(a, c)。显然a >= c，则f(x) = c。

ll count(ll n) {
    ll res = 0;
    while(n) {
        res += n / 5;
        n /= 5;
    }
    return res;
}

n/5表示5的倍数贡献一个5，n/25表示25的倍数再贡献一个，以此类推

二、使用二分查找

通过二分查找我们得到f(x)等于K的最大的x值，如果没有则返回最右侧位置。这得益于使用了count(mi) > k。每次迭代lo - 1的位置是当前确定的小于等于e的最大元素，hi位置是当前确定的大于e的最小元素。当结束时，lo = hi，lo-1即为所找元素。若改用 >= 号，返回的是最左侧的位置。

三、做差

f(x) = K的最大x值与f(x) = K - 1的最大x值即为f(x) = K的元素个数。
